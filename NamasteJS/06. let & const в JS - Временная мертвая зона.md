````markdown
# 1. Введение в концепции

## Ключевые вопросы:

- Что такое временная «мертвая зона» (Temporal Dead Zone, TDZ)?
- Поднимаются ли объявления с `let` и `const` (hoisted)?

- В чём разница между синтаксической ошибкой (SyntaxError),
  ошибкой ссылки (ReferenceError) и ошибкой типа (TypeError)?

## Основная мысль:

Объявления через `let` и `const` действительно поднимаются, но ведут себя иначе, чем `var`.
Они входят в состояние, известное как временная «мертвая зона» (TDZ),
до тех пор, пока не будет выполнено их объявление.

---

# 2. Hoisting: let/const vs. var

## A. Объявления через var

### Поведение:

- При использовании `var` JavaScript выделяет память во время фазы создания контекста выполнения.
- Переменные, объявленные через `var`, автоматически инициализируются значением `undefined`.
- Они привязываются к глобальному объекту (например, `window` в браузерах).

### Пример:

```javascript
console.log(b); // Выведет: undefined (без ошибки)
var b = 100;
console.log(b); // Выведет: 100
console.log(window.b); // Выведет: 100 (в браузерах)
```
````

## B. Объявления через let и const

### Поведение:

- Они тоже поднимаются (hoisted), память выделяется во время фазы создания.
- Однако, в отличие от `var`, они не инициализируются сразу.
  Вместо этого они находятся в состоянии, когда доступ к ним невозможен до тех пор,
  пока не будет выполнено объявление.

- Этот период называется Temporal Dead Zone (TDZ).
- Они хранятся в отдельном пространстве памяти (и не привязываются к глобальному объекту).

### Пример (let):

```javascript
console.log(a); // Ошибка ReferenceError: Cannot access 'a' before initialization
let a = 10;
console.log(a); // Теперь выведет: 10
console.log(window.a); // В браузерах выведет: undefined
```

### Пример (const):

```javascript
// Аналогичное поведение hoisting, как у let:
console.log(c); // ReferenceError: Cannot access 'c' before initialization
const c = 20;
console.log(c); // Выведет: 20

// Также, const должен быть инициализирован сразу при объявлении:
// const d;  // SyntaxError: Missing initializer in const declaration
```

# 3. Временная «мертвая зона» (TDZ)

## Определение:

TDZ — это период между тем, как для переменной выделяется память (hoisting),
и тем, как выполняется её объявление (инициализация значением).

## Ключевые характеристики:

### Во время TDZ:

- Переменная существует в памяти, но к ней нельзя обратиться.
- Попытка доступа приводит к ошибке `ReferenceError`.

### После инициализации:

Как только выполнение кода доходит до строки, где переменной присваивается значение,
TDZ заканчивается, и переменная становится доступной.

## Пример кода, иллюстрирующий TDZ:

```javascript
// TDZ начинается с начала блока до выполнения объявления let:
{
  // На этом этапе переменная 'x' поднята, но не инициализирована:
  console.log(x); // ReferenceError: Cannot access 'x' before initialization
  let x = 50; // Здесь x инициализируется; TDZ заканчивается
  console.log(x); // Выведет: 50
}
```

## Практический совет:

Чтобы избежать проблем, связанных с TDZ, всегда объявляйте и инициализируйте
ваши переменные `let`/`const` в начале их области видимости.

# 4. Типы ошибок: Syntax, Reference и Type Errors

## A. Синтаксическая ошибка (SyntaxError)

### Когда возникает:

- Во время стадии парсинга, до выполнения кода.
- Вызывается, когда код нарушает правила языка.

### Примеры:

**Повторное объявление переменной с использованием let в одной области:**

```javascript
let a = 10;
let a = 20; // SyntaxError: Identifier 'a' has already been declared
```

**Объявление const без инициализации:**

```javascript
const b; // SyntaxError: Missing initializer in const declaration
```

## B. Ошибка ссылки (ReferenceError)

### Когда возникает:

Когда код пытается обратиться к переменной, которая либо:

- Не существует ни в одной доступной области (например, переменная не объявлена вовсе), либо
- Находится в TDZ (поднята, но ещё не инициализирована).

### Примеры:

**Обращение к переменной в TDZ:**

```javascript
console.log(x); // ReferenceError: Cannot access 'x' before initialization
let x = 10;
```

**Обращение к необъявленной переменной:**

```javascript
console.log(y); // ReferenceError: y is not defined
```

## C. Ошибка типа (TypeError)

### Когда возникает:

- При выполнении операции над значением, которое не соответствует ожидаемому типу.
- Часто возникает при попытке изменить значение переменной, объявленной через `const`.

### Примеры:

**Попытка переназначить переменную const:**

```javascript
const z = 30;
z = 40; // TypeError: Assignment to constant variable.
```

**Попытка вызвать не функцию:**

```javascript
const foo = 123;
foo(); // TypeError: foo is not a function
```

# 5. Лучшие практики и практические советы

## Предпочтения при объявлении переменных:

### Используйте const в первую очередь:

Применяйте `const` для переменных, которые не изменяются.
Это помогает избежать случайного переназначения.

### Затем используйте let:

Применяйте `let` для переменных, которые предполагается переназначать.

### Избегайте var, где это возможно:

`var` менее предсказуем, так как он поднимается (hoisted) и привязывается к глобальному объекту.

## Как избежать проблем с TDZ:

**Совет:** Объявляйте и инициализируйте переменные в начале их области видимости.
Это «сжимает» временную мёртвую зону почти до нуля,
снижая риск случайного доступа к неинициализированной переменной.

## Ответы на вопросы для собеседований:

Если вас спросят, поднимаются ли `let`/`const`, объясните, что они поднимаются,
но остаются неинициализированными (находятся в TDZ) до тех пор,
пока выполнение кода не дойдёт до их объявления.
Именно поэтому доступ к ним до объявления приводит к `ReferenceError`.

# 6. Резюме и заключительные мысли

## Резюме по hoisting:

- **var:** Поднимается, автоматически инициализируется значением `undefined`, привязан к глобальному объекту.
- **let/const:** Поднимаются, но остаются неинициализированными (применяется TDZ);
  хранятся в отдельном пространстве и не привязываются к глобальному объекту.

## Временная мёртвая зона (TDZ):

Это период от начала области видимости до момента выполнения строки объявления переменной.
Попытка доступа к переменной `let`/`const` в этот период приводит к `ReferenceError`.

## Типы ошибок:

- **SyntaxError:** Проблемы со структурой кода (например, дублирование объявлений, отсутствие инициализатора для const).
- **ReferenceError:** Возникает при попытке доступа к переменной, находящейся в TDZ или не объявленной вообще.
- **TypeError:** Возникает, когда выполняется операция над несовместимым типом (например, переназначение const).

## Лучшие практики:

- По возможности используйте `const`, а затем `let`, если требуется возможность переназначения.
- Держите объявления переменных в начале области видимости, чтобы минимизировать проблемы с TDZ.

Понимание этих концепций и их взаимодействия в JavaScript поможет вам
эффективнее отлаживать код и уверенно отвечать на вопросы на собеседованиях.
