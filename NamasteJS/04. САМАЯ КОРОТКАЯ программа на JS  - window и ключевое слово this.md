````markdown
# 1. «Самая короткая» программа на JavaScript

## Утверждение:

Спикер задаёт вопрос:

«Знаете, какая самая короткая программа на JavaScript?»
**Ответ:** Пустой файл.

## Почему это интересно?

Даже если файл пустой, движок JavaScript за кулисами выполняет кучу работы.

---

# 2. За кулисами пустого файла

## Глобальный Контекст Выполнения (GEC):

### Что происходит:

При запуске даже пустого файла движок JS создаёт **Глобальный Контекст Выполнения**.
Этот контекст — это «контейнер», в котором будет исполняться весь код, если он появится.

### Выделение памяти:

Движок подготавливает область памяти
(даже если туда ничего не записано) для переменных, функций и прочего.

## Глобальный объект:

### Определение:

В браузерах глобальный объект называется `window`.
Он создаётся автоматически движком JS вместе с Глобальным Контекстом Выполнения.

### Пример:

Если ввести в консоли браузера `window`,
вы увидите огромный объект, полный свойств и методов.

## Ключевое слово this:

На глобальном уровне `this` указывает на глобальный объект.

### Пример:

```javascript
console.log(this === window); // true (в браузере)
```
````

# 3. Глобальный объект и глобальная область видимости

## Глобальная область видимости (Global Scope):

### Определение:

Любой код, который не обёрнут в функцию,
принадлежит глобальной области видимости.

### Что происходит:

Когда вы объявляете переменные или функции на верхнем уровне,
они становятся свойствами глобального объекта.

### Пример:

```javascript
// Объявление глобальной переменной
var a = 10;

// Объявление глобальной функции
function b() {
  console.log("Я в глобальной области видимости!");
}

// 'a' и 'b' становятся свойствами глобального объекта
console.log(window.a); // 10
console.log(window.b); // function b() { ... }
```

## Доступ к переменным:

При обращении к глобальной переменной (например, `a`)
JavaScript ищет её в глобальном объекте.
Если переменная не была объявлена, возникает ошибка «not defined».

# 4. Демонстрация с использованием отладчика и консоли

## Использование отладчика:

Спикер предлагает поставить `debugger` в самом начале файла.

### Цель:

Остановить выполнение до начала кода,
чтобы можно было заглянуть в глобальную память.

### Что вы увидите:

- До выполнения первой строки уже создан Глобальный Контекст Выполнения.
- Память содержит глобальный объект (`window`) и все зарезервированные места
  (например, переменные, объявленные позже, с начальным значением `undefined`).

## Наблюдения в консоли:

### При наличии объявления:

```javascript
var a = 10;

// В консоли:
console.log(window.a); // 10
console.log(a); // 10 (так как a находится в глобальной области)
```

### Когда переменная не объявлена:

```javascript
// Если переменная x никогда не объявлялась:
console.log(x); // ReferenceError: x is not defined
```

## Роль ключевого слова this:

```javascript
console.log(this.a); // Выведет 10, так как this === window на глобальном уровне.
```

# 5. Резюме ключевых моментов

## Самая короткая программа:

Пустой файл — это самая короткая программа на JavaScript,
но даже он инициирует создание Глобального Контекста Выполнения и Глобального Объекта.

## Глобальный Контекст Выполнения и настройка памяти:

Даже без явного кода движок подготавливает:

- Глобальный Контекст Выполнения.
- Область памяти для переменных и функций (если они будут объявлены).

## Глобальный объект (window в браузерах):

Создаётся автоматически движком JS и содержит:

- Стандартные свойства, методы,
- А также любые глобальные переменные и функции, которые вы создаёте.

## Ключевое слово this:

На глобальном уровне `this` указывает на глобальный объект.

## Глобальная vs. Локальная область видимости:

### Глобальная область видимости:

- Всё, что написано вне функции.
- Переменные и функции здесь становятся свойствами глобального объекта.

### Локальная область видимости:

- Код внутри функции работает в своём отдельном Контексте Выполнения
- И имеет собственную память, не привязанную к глобальному объекту.

## Напоминание про hoisting:

Да, это не основное здесь, но помните: переменные, объявленные через `var`,
поднимаются (hoisting) и получают начальное значение `undefined` — это часть глобального процесса выполнения.

# 6. Заключительные мысли и дальнейшие шаги

## Почему это важно:

Понимание того, что даже «пустая» программа создаёт полную среду выполнения,
помогает осознать, что движок JavaScript всегда занят подготовкой сцены
перед выполнением вашего кода.

Это базовые знания о Глобальном Контексте Выполнения,
Глобальном Объекте и ключевом слове `this`, которые крайне важны для отладки
и эффективного программирования на JavaScript.
