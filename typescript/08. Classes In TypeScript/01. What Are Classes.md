### Объяснение концепции классов

Автор начинает новый большой раздел курса, посвященный **классам в TypeScript**. Давайте пройдемся по каждому его утверждению шаг за шагом.

---

#### 1. "TypeScript classes are an extension of JavaScript classes itself, but they are different in certain ways."

**Что это значит:**

- **"Расширение JavaScript классов"**: TypeScript не изобретает классы заново. Синтаксис классов (ключевое слово `class`, конструкторы, методы и т.д.) пришел из современного JavaScript (ES6 и выше). Это означает, что валидный класс TypeScript часто является и валидным классом JavaScript.
- **"Но они отличаются определенным образом"**: Здесь TypeScript добавляет свою "магию" — систему типов. Основные отличия, которые вы скоро изучите, включают:
  - **Аннотации типов для свойств и методов**: Вы можете явно указать, какого типа данные может хранить свойство класса или какие аргументы и возвращаемое значение у метода.
  - **Модификаторы доступа**: `public`, `private`, `protected` — эти ключевые слова позволяют контролировать, откуда можно получить доступ к свойствам и методам класса (извне объекта, только из самого класса, из классов-наследников).
  - **Абстрактные классы и методы**: Возможность создавать шаблоны классов, которые не могут быть instantiated (созданы как объекты) напрямую, а только через наследование.

**Простая аналогия:** Представьте, что JavaScript — это базовый автомобиль. TypeScript — это тот же автомобиль, но с дополнительным набором инструментов для навигации и предупреждений (система типов), который делает вождение более безопасным и предсказуемым.

---

#### 2. "A class is a template or a blueprint of creating objects."

Это ключевое определение. Автор очень тщательно его разжевывает, и правильно делает.

**Разберем аналогию "Шаблон" или "Чертеж":**

- **Класс = Чертеж здания.** На чертеже указано, что в здании должен быть фундамент, стены, окна, двери, крыша. Но сам чертеж — это не здание. Вы не можете жить в чертеже.
- **Объект = Реальное здание, построенное по этому чертежу.** Это конкретный экземпляр с конкретными характеристиками: стены покрашены в синий цвет, окна из стеклопакета, дверь дубовая.

Автор приводит пример с пользователями (`user1` и `user2`). Оба объекта имеют одинаковую структуру (например, свойства `firstName`, `lastName`, `email`), но разные значения (`John Doe` vs `Mark Smith`).

**Проблема, которую решают классы:**
Без класса вам пришлось бы каждый раз вручную создавать каждый такой объект:

```typescript
// Без класса (ручное создание объектов)
const user1 = {
  firstName: "John",
  lastName: "Doe",
  email: "john@example.com",
  greet() {
    console.log(`Hello, ${this.firstName}!`);
  },
};

const user2 = {
  firstName: "Mark",
  lastName: "Smith",
  email: "mark@example.com",
  greet() {
    console.log(`Hello, ${this.firstName}!`);
  },
};
```

Видите проблему? Код повторяется. Если мы захотим добавить новое свойство (например, `age`), нам придется обходить все такие объекты и добавлять его вручную. Это неэффективно и чревато ошибками.

---

#### 3. "A class can be considered as a blueprint... you just create a new instance of this particular class."

Здесь автор вводит терминологию.

- **Создание класса (blueprint):** Вы один раз описываете структуру.
- **Создание экземпляра (instance):** Вы используете этот класс для создания конкретного объекта. Этот процесс называется **инстанцированием**.

**Как это выглядит в коде (как намекает автор):**

```typescript
// 1. Объявляем класс (чертеж)
class User {
  name: string;
  email: string;

  // Это конструктор - специальный метод для инициализации нового объекта.
  constructor(name: string, email: string) {
    this.name = name;
    this.email = email;
  }

  // Метод класса
  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }
}

// 2. Создаем экземпляры (реальные объекты) с помощью ключевого слова "new"
const user1 = new User("John", "john@example.com");
const user2 = new User("Mark", "mark@example.com");

// user1 и user2 - это два разных объекта, созданных по одному "чертежу" User.
// Они имеют одинаковую структуру (свойства name, email, метод greet),
// но содержат разные данные.

user1.greet(); // Вывод: "Hello, my name is John"
user2.greet(); // Вывод: "Hello, my name is Mark"
```

**Мышление "Фабрикой":**
Автор говорит: "класс можно рассматривать как фабрику". Это еще одна прекрасная аналогия. Класс `User` — это фабрика по производству пользователей. Каждый раз, когда вы пишете `new User(...)`, вы "запускаете конвейер" и на выходе получаете нового, готового пользователя, укомплектованного всеми необходимыми свойствами и методами.

### Итог лекции

Автор подготовил почву для практического изучения классов. Он объяснил **"почему"** (избежание повторяющегося кода, создание согласованных объектов) и **"что"** (класс — это чертеж для создания объектов).

**Ключевые выводы, которые вы должны усвоить перед следующей лекцией:**

1.  **Класс — это не объект.** Это план, по которому создаются объекты.
2.  **Основная цель класса** — создание множества объектов с одинаковой структурой, но разными данными.
3.  **Процесс создания объекта из класса** называется созданием экземпляра (instantiation) и использует ключевое слово `new`.
4.  **TypeScript расширяет классы JavaScript**, добавляя аннотации типов и другие функции для большей надежности кода.

В следующем видео автор, скорее всего, покажет, как именно записать этот "чертеж" в коде на TypeScript, то есть как объявить свой первый класс. Вы готовы к этому переходу от теории к практике.
