## Объяснение конструктора в TypeScript классе

### 1. Проблема статического класса

В предыдущей лекции мы создали класс, который всегда создает одинаковые объекты:

```typescript
class User {
  name = "John";
  email = "john@email.com";

  greet() {
    return "Hello John";
  }
}

const user1 = new User(); // Всегда John
const user2 = new User(); // Всегда John
```

**Проблема:** Невозможно создать пользователей с разными именами и email.

### 2. Введение конструктора

**Конструктор** — это специальная функция в классе, которая:

- Вызывается автоматически при создании нового объекта (при использовании `new`)
- Позволяет принимать параметры при создании объекта
- Использует ключевое слово `constructor`

```typescript
class User {
  name = "John";
  email = "john@email.com";

  // Конструктор - добавляем параметры
  constructor(name: string, email: string) {
    console.log(name);
    console.log(email);
  }

  greet() {
    return "Hello John";
  }
}
```

### 3. Как работает конструктор

**Процесс создания объекта:**

1. `const user = new User("John", "john@email.com");`
2. Автоматически вызывается `constructor("John", "john@email.com")`
3. Выполняется код внутри конструктора
4. Создается объект

Автор демонстрирует это с помощью `console.log`:

```typescript
constructor(name: string, email: string) {
    console.log(name);    // "John" для первого пользователя
    console.log(email);   // "john@email.com" для первого пользователя
}
```

При создании двух пользователей:

```typescript
const user1 = new User("John", "john@email.com");
// В консоли: "John", "john@email.com"

const user2 = new User("Mark", "mark@email.com");
// В консоли: "Mark", "mark@email.com"
```

### 4. Ошибка TypeScript

После добавления параметров в конструктор TypeScript начинает требовать передачи аргументов:

```typescript
// ОШИБКА: Expected 2 arguments, but got 0
const user = new User();

// ПРАВИЛЬНО: передаем два аргумента
const user = new User("John", "john@email.com");
```

**Почему это происходит:** TypeScript анализирует конструктор и понимает, что для создания объекта нужны два аргумента типа string.

### 5. Текущее состояние проблемы

Хотя конструктор теперь принимает параметры, поля класса все еще остаются статическими:

```typescript
class User {
  name = "John"; // Всегда "John"
  email = "john@email.com"; // Всегда "john@email.com"

  constructor(name: string, email: string) {
    console.log(name); // Выводим переданные значения
    console.log(email);
    // НО поля name и email класса не меняются!
  }
}

const user1 = new User("Alice", "alice@email.com");
console.log(user1.name); // "John" (а не "Alice!")
```

**Проблема:** Параметры конструктора не присваиваются полям класса.
