## Создание первого класса в TypeScript

### 1. Цель создания класса

Автор хочет создать класс `User`, который будет **шаблоном** для создания объектов пользователей с тремя компонентами:

- **name** - имя пользователя
- **email** - email пользователя
- **greet()** - метод для приветствия

### 2. Объявление класса

```typescript
class User {
  // содержимое класса
}
```

**Ключевые моменты:**

- `class` - ключевое слово для создания класса
- `User` - имя класса (по соглашению с большой буквы)
- **Класс ≠ объект** - класс это шаблон, а объект - конкретная реализация

### 3. Поля класса (Fields)

```typescript
class User {
  name = "John";
  email = "john@email.com";
}
```

**Важная терминология:**

- **Поле (Field)** - переменная в классе (в контексте TypeScript)
- **Свойство (Property)** - то же самое, но когда говорим об объекте
- **Метод (Method)** - функция в классе

### 4. Добавление метода

```typescript
class User {
  name = "John";
  email = "john@email.com";

  greet() {
    return "Hello John";
  }
}
```

**Особенности методов в классах:**

- Не используется ключевое слово `function`
- Метод автоматически становится частью объектов, созданных из класса

### 5. Создание экземпляра класса

```typescript
const user = new User();
console.log(user);
```

**Процесс инстанцирования:**

- `new` - ключевое слово для создания нового объекта из класса
- `User()` - вызов класса как функции (на самом деле вызывается конструктор)
- **Экземпляр (instance)** - конкретный объект, созданный из класса

### 6. Что происходит при создании объекта

Когда вы создаете `new User()`, получается объект со структурой:

```javascript
{
    name: "John",
    email: "john@email.com"
    // метод greet находится в прототипе
}
```

**Прототипы (объяснение просто):**

- Методы помещаются в **прототип** для экономии памяти
- Все объекты одного класса используют одни и те же методы из прототипа
- Вы все равно можете вызывать `user.greet()` - это работает!

### 7. Вызов метода

```typescript
console.log(user.greet()); // "Hello John"
```

### 8. Проблема текущей реализации

Автор показывает главную проблему:

```typescript
const user1 = new User();
const user2 = new User();
// ОБА объекта будут иметь:
// name = "John"
// email = "john@email.com"
// greet() возвращает "Hello John"
```

**Проблемы:**

1. **Все объекты идентичны** - нельзя создать пользователей с разными именами
2. **Статические значения** - значения жестко заданы в классе
3. **Негибкость** - класс не принимает параметры при создании

### 9. Индикация в консоли

Когда вы выводите `user` в консоль, браузер показывает:

```
User {name: "John", email: "john@email.com"}
```

Это указывает, что объект является **экземпляром класса User**, а не обычным объектом.

### Итог текущего состояния

**Что мы имеем:**

- ✅ Создан первый класс
- ✅ Добавлены поля и метод
- ✅ Научились создавать объекты из класса
- ✅ Понимаем разницу между классом и объектом

**Что нужно исправить (в следующей лекции):**

- ❌ Сделать класс динамическим
- ❌ Добавить возможность передавать разные значения при создании
- ❌ Решить проблему одинаковых объектов

### Ключевые концепции, которые нужно запомнить:

1. **Класс = шаблон**, **Объект = экземпляр**
2. **Поля** - данные класса, **Методы** - функции класса
3. **`new ClassName()`** - создание нового объекта
4. **Прототипы** - механизм JavaScript для хранения методов
5. **Статичность** - текущая проблема, которую нужно решить
