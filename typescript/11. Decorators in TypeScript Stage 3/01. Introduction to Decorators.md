## 1. Введение в раздел о декораторах

Автор начинает новый раздел курса, посвященный **декораторам** (decorators). Важно отметить, что в курсе будет **два раздела** о декораторах:

- **Текущий раздел**: TypeScript 5+ декораторы (stage 3 декораторы)
- **Следующий раздел**: TypeScript 4- декораторы (stage 2/экспериментальные декораторы)

## 2. Что такое декоратор?

**Декоратор** - это специальная функция в TypeScript, которая может:

- **Модифицировать** классы, методы, свойства или параметры
- **Использоваться только с классами** и их элементами

### Ключевые характеристики:

- Это **специальная функция-декларация**
- Работает **только с классами** (не с обычными функциями или объектами)
- Может применяться к:
  - Самому классу
  - Методам класса
  - Свойствам класса
  - Аксессорам (get/set методам)

## 3. Синтаксис декораторов

### Объявление декоратора:

```typescript
// Это функция-декоратор
function doSomething() {
  // логика декоратора
}
```

### Применение декоратора:

```typescript
@doSomething
class MyClass {
  // применение к классу
}
```

## 4. Различия между версиями декораторов

### Stage 2 декораторы (TypeScript 4 и ранее):

- **Экспериментальные** декораторы
- Поддерживают **параметры методов**
- Имеют **Reflection API** и метаданные
- Используются в фреймворках (Angular, NestJS)

### Stage 3 декораторы (TypeScript 5+):

- **Современные** декораторы
- **Не поддерживают** параметры методов
- **Выровнены со стандартом ECMAScript**
- **Не имеют** встроенной поддержки метаданных

## 5. Настройки TypeScript для работы с декораторами

В `tsconfig.json` есть два ключевых флага:

```json
{
  "compilerOptions": {
    // Для Stage 2 декораторов (старых):
    // "experimentalDecorators": true,
    // "emitDecoratorMetadata": true,
    // Для Stage 3 декораторов (новых):
    // эти флаги должны быть false или отсутствовать
  }
}
```

### Важные моменты:

- Если `experimentalDecorators: true` - используются **старые** декораторы
- Если флаги отключены - используются **новые** декораторы TypeScript 5
- **Проверяйте эти настройки**, если результаты не совпадают с ожидаемыми

## 6. Практическая настройка окружения

Автор показывает настройку проекта:

1. **Создает TypeScript файл** (`app.ts`) вместо JavaScript
2. **Настраивает HTML** для подключения скомпилированного файла:
   ```html
   <script src="dist/app.js"></script>
   ```
3. **Проверяет tsconfig.json** - что экспериментальные декораторы отключены
4. **Запускает Live Server** для разработки

## 7. Основные выводы для начала работы

1. **Декораторы** - мощный инструмент для модификации классов
2. **TypeScript 5** принес новые декораторы, совместимые со стандартом JavaScript
3. **Старые проекты** могут требовать старых декораторов через флаги в tsconfig
4. **Начинаем изучение** с современных декораторов TypeScript 5
