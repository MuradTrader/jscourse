### Общий смысл видео

Автор объясняет разницу между двумя типами, которые указывают, что функция не возвращает значение: `void` (функция завершается, но ничего не возвращает) и `never` (функция никогда не завершается нормально).

---

### Подробное объяснение по пунктам

#### 1. Тип `void` - функция не возвращает значение

**Что говорит автор:**

> "There are times when your functions do not return anything... There is a type available and this type is called void. Void is a special TypeScript type which is used with functions that do not return anything."

**Что это значит, подробно:**

- **`void`** используется для функций, которые **выполняются до конца**, но не возвращают никакого значения через `return`.
- Такие функции могут выполнять side-effects (побочные эффекты): вывод в консоль, запись в базу данных, изменение DOM и т.д.
- В JavaScript такие функции технически возвращают `undefined`, но TypeScript рассматривает это как отдельный тип `void`.

```typescript
// Функция завершается, но не возвращает значение
function writeToDatabase(value: string): void {
  console.log(`Writing to database: ${value}`);
  // Нет оператора return - функция завершается, возвращая undefined
}

// Это эквивалентно:
function writeToDatabase(value: string): void {
  console.log(`Writing to database: ${value}`);
  return; // Пустой return - тоже void
  // или return undefined; - тоже void
}
```

#### 2. Тип `never` - функция никогда не завершается

**Что говорит автор:**

> "The difference between void and never is that you assign a never type to a function that never completes its execution. So especially to a function that throws an error."

**Что это значит, подробно:**

- **`never`** используется для функций, которые **никогда не завершают свое выполнение** нормальным образом.
- Такие функции либо:
  - Выбрасывают исключение (`throw`)
  - Выполняют бесконечный цикл
  - Приводят к завершению программы

```typescript
// Функция никогда не завершается нормально - всегда выбрасывает ошибку
function throwError(error: string): never {
  throw new Error(error);
  // Код после throw никогда не выполнится
}

// Другие примеры never-функций:
function infiniteLoop(): never {
  while (true) {
    // Бесконечный цикл - функция никогда не завершится
  }
}
```

#### 3. Ключевые различия между `void` и `never`

**Что говорит автор:**

> "void function over here would complete its execution. It just does not return anything. Never, however, would never complete the execution."

**Что это значит, подробно:**

| Аспект            | `void`                             | `never`                                |
| ----------------- | ---------------------------------- | -------------------------------------- |
| **Завершение**    | Функция завершается нормально      | Функция никогда не завершается         |
| **Возврат**       | Не возвращает значение (undefined) | Не возвращает ничего (даже undefined)  |
| **Использование** | Side-effects, логирование          | Обработка ошибок, бесконечные процессы |

```typescript
// void - выполняется, но не возвращает значение
function logMessage(message: string): void {
  console.log(message); // Выполняется
  // Функция завершается
}

// never - никогда не завершается
function crashProgram(): never {
  throw new Error("Program crashed!"); // Выполняется
  // Функция НИКОГДА не завершается нормально
  console.log("This will never run"); // Этот код недостижим!
}
```

#### 4. Иерархия типов: `never` extends `void`

**Что говорит автор:**

> "Never is a subtype of void... you cannot use never instead of void. TypeScript will throw an error."

**Что это значит, подробно:**

- **`never` является подтипом `void`** - это значит, что везде, где ожидается `void`, можно использовать `never`, но не наоборот.
- **Аналогия:** Представьте, что `void` - это "транспортное средство", а `never` - "скорая помощь". Везде, где разрешены транспортные средства, можно использовать скорую помощь. Но там, где требуется именно скорая помощь, нельзя использовать любое транспортное средство.

```typescript
// Правильно: never можно использовать вместо void
function example1(): void {
  throw new Error(); // OK - never совместим с void
}

// НЕПРАВИЛЬНО: void нельзя использовать вместо never
function example2(): never {
  console.log("Hello"); // ОШИБКА! Эта функция может завершиться нормально
}
```

#### 5. Практическая демонстрация отношений типов

**Что говорит автор:**

> "Let's check whether never extends void or not... Check is set to true... Check two is false."

**Что это значит, подробно:**

TypeScript позволяет проверять отношения типов с помощью условных типов:

```typescript
// Проверяем: является ли never подтипом void?
type Check = never extends void ? true : false;
// Результат: true - never действительно подтип void

// Проверяем: является ли void подтипом never?
type Check2 = void extends never ? true : false;
// Результат: false - void НЕ является подтипом never
```

#### 6. Когда использовать каждый тип

**Что говорит автор:**

> "Only use never when you are sure that your function execution will never complete... for any other function that does not return anything... use void."

**Практические рекомендации:**

**Используйте `void` для:**

- Функций логирования
- Функций сохранения данных
- Обработчиков событий
- Любых функций, которые выполняются до конца, но не возвращают значение

**Используйте `never` для:**

- Функций, выбрасывающих исключения
- Бесконечных циклов (например, в играх)
- Функций, которые всегда прерывают выполнение программы

---

### Ключевые выводы из урока

1.  **`void`** - для функций, которые **завершаются**, но не возвращают значение
2.  **`never`** - для функций, которые **никогда не завершаются** нормально
3.  **Иерархия:** `never` является подтипом `void` - можно использовать `never` там, где ожидается `void`, но не наоборот
4.  **Проверка типов:** TypeScript строго следит за правильным использованием этих типов
5.  **Практика:** Используйте соответствующий тип в зависимости от поведения функции
