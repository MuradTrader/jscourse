### Общий смысл видео

В этом видео автор объясняет две фундаментальные концепции параметров функций в TypeScript: **обязательные (default)** и **опциональные (optional) параметры**. Он показывает, чем они отличаются, как их объявлять и каковы правила их совместного использования.

---

### Подробное объяснение по пунктам

#### 1. Обязательные параметры (Default Parameters)

**Что говорит автор:**

> "Default parameters are those parameters which are required by your function... you cannot invoke this intro function by just passing in one argument. You have to pass in both the arguments."

**Что это значит, подробно:**

- **"Default" может сбить с толку.** В контексте этого урока автор использует термин "default parameters" не в том смысле, к которому мы придем позже (параметры по умолчанию, например `function greet(name: string = "Аноним")`). Здесь он имеет в виду **"параметры по умолчанию" в смысле "стандартные, обязательные"**. Более точный технический термин для этого — **обязательные параметры (required parameters)**.
- **Суть обязательных параметров:** Когда функция объявлена с такими параметрами (например, `name` и `age`), при ее вызове **обязательно** нужно передать соответствующие аргументы для каждого из них. TypeScript следит за этим на этапе компиляции.
- **Пример из жизни:** Представьте, что вы заполняете анкету для получения паспорта. Поля "Фамилия" и "Имя" являются обязательными. Вы не можете отправить анкету, заполнив только одно из них.

```typescript
// Объявление функции с обязательными параметрами
function intro(name: string, age: number): string {
  return `My name is ${name} and I am ${age} years old.`;
}

// Правильный вызов
const message = intro("John", 30); // ОК

// Неправильный вызов - TypeScript выдаст ошибку
const errorMessage = intro("John"); // ОШИБКА: Expected 2 arguments, but got 1.
```

#### 2. Опциональные параметры (Optional Parameters)

**Что говорит автор:**

> "There are times when you need optional parameters as well... I want this to be optional. I don't want to have country for every function invocation."

**Что это значит, подробно:**

- **Гибкость для функций.** Бывают ситуации, когда некоторым параметрам не нужно присваивать значение при каждом вызове функции. Они являются необязательными.
- **Синтаксис:** Чтобы сделать параметр опциональным, после его имени ставится **знак вопроса (`?`)**.
  - `function intro(name: string, age: number, country?: string) { ... }`
- **Тип опционального параметра:** Когда вы делаете параметр опциональным, TypeScript неявно добавляет к его типу `undefined`. То есть `country?: string` это то же самое, что `country: string | undefined`. Это важно понимать, потому что вы не можете использовать `undefined` как значение, если не указали его в типе.
- **Поведение внутри функции:** Поскольку опциональный параметр может быть не передан, его значение внутри функции может быть `undefined`. Автор показывает это, изменяя тело функции и добавляя проверку (`if (country) { ... }`). Это стандартная практика работы с опциональными параметрами.

```typescript
function intro(name: string, age: number, country?: string): string {
  // Проверяем, был ли передан country
  if (country) {
    return `My name is ${name}, I am ${age} years old, and I live in ${country}.`;
  }
  // Если country не передан, используем этот вариант
  return `My name is ${name} and I am ${age} years old.`;
}

// Оба варианта вызова корректны
const message1 = intro("John", 30); // Страна не указана
const message2 = intro("John", 30, "USA"); // Страна указана
```

#### 3. Важнейшее правило порядка следования параметров

**Что говорит автор:**

> "Also, please note, optional parameters can only be declared after the default parameters have been declared. You cannot declare an optional parameter before the default parameter."

**Что это значит, подробно:**

- **Это правило абсолютно.** Параметры в функции должны следовать в строгом порядке:
  1.  **Сначала** все обязательные параметры.
  2.  **Потом** все опциональные параметры.
  3.  _(Позже вы изучите) **В самом конце** — параметры rest (`...rest: type[]`)._
- **Почему это правило существует?** Потому что аргументы функции передаются по позиции. Представьте, что вы вызываете функцию `intro("John", "USA")`. Как TypeScript (и JavaScript) поймет, что вы имели в виду? `age` пропущен, а `country` передан? Или `age` равен "USA"? Это привело бы к неоднозначности. Правило гарантирует, что позиции обязательных аргументов всегда фиксированы и предсказуемы.

**Неправильно:**

```typescript
function intro(country?: string, name: string, age: number): string {
  // ОШИБКА!
  // Required parameters must follow optional parameters.
  // ... тело функции
}
```

**Правильно:**

```typescript
function intro(name: string, age: number, country?: string): string {
  // OK
  // ... тело функции
}
```

---

### Итоговый конспект урока

1.  **Обязательные параметры:** Параметры, которые должны быть переданы в функцию при ее вызове. TypeScript выдаст ошибку, если какой-либо из обязательных аргументов будет пропущен.
2.  **Опциональные параметры:** Параметры, которые можно передать, а можно и пропустить. Обозначаются символом `?` после имени. Их тип автоматически становится `type | undefined`.
3.  **Правило порядка:** Сначала в списке параметров объявляются все обязательные, и только после них — опциональные. Нарушение этого правила приводит к ошибке компиляции.
