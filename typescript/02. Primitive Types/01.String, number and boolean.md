### Общее введение

Автор начинает с трех простейших примитивных типов: **string** (строка), **number** (число) и **boolean** (логический тип). Весь код пишется в файле `index.ts`. Это стандартная практика для начала изучения.

---

### 1. Тип `string` (Строка)

**Что такое строка?**
В TypeScript, как и в JavaScript, строка — это последовательность символов, используемая для представления текста. Например, `"John"`, `"Hello World"`.

**Пример от автора:**

```typescript
let firstName: string = "John";
```

- `let` — ключевое слово для объявления переменной.
- `firstName` — имя переменной.
- `: string` — это **аннотация типа** (type annotation). Мы вручную указываем TypeScript, что в этой переменной можно хранить только строки.
- `= "John"` — присвоение значения.

**Ключевой момент:** После такой аннотации TypeScript будет следить за типом переменной. Если вы попытаетесь присвоить ей значение другого типа, компилятор выдаст ошибку.

```typescript
firstName = 42; // ОШИБКА: Тип 'number' не может быть присвоен типу 'string'
```

**`let` vs `var`**
Автор упоминает, что и `let`, и `var` можно использовать с аннотациями типов. Поведение в плане типизации будет одинаковым. Разница между `let` и `var` касается области видимости (scope), но это уже особенность JavaScript, а не TypeScript.

---

### 2. Важные концепции: Аннотация (Annotation) и Вывод (Inference)

**Аннотация (Type Annotation)**
Это когда вы _явно_ указываете тип переменной, используя синтаксис `: Type`. Автор говорит, что будет часто использовать этот термин в курсе.

**Вывод типа (Type Inference)**
Это одна из самых мощных возможностей TypeScript. Компилятор не всегда требует явной аннотации. Он умеет _"догадываться"_ (infer) о типе переменной на основе её значения.

**Пример:**

```typescript
let automobile = "Car"; // Мы не указали тип явно.
```

Если вы наведете курсор на `automobile` в вашей IDE, вы увидите подсказку: `let automobile: string`. TypeScript проанализировал значение `"Car"` и самостоятельно определил, что тип переменной — `string`.

**Зачем нужны аннотации, если есть вывод?**
Автор обещает раскрыть это позже, но вот краткий ответ:

- **Читаемость кода:** Аннотации делают код более понятным для других разработчиков.
- **Контроль над типом:** Вы можете захотеть, чтобы у переменной был тип `string`, даже если она инициализируется пустой строкой, которую потом заполните.
- **Сложные случаи:** Вывод не всегда может определить тип точно, особенно в сложных сценариях.

### 3. Особенности с `const` (Константой)

Когда вы объявляете переменную с помощью `const`, она не может быть перезаписана. TypeScript использует это знание для более точного вывода типа.

```typescript
const city = "New York";
```

В этом случае TypeScript выведет не просто тип `string`, а **литеральный тип** `"New York"`. Это значит, что для TypeScript `city` — это не любая строка, а строка со значением `"New York"`. Это часть системы типов TypeScript, обеспечивающая повышенную типобезопасность.

**Примитивные типы иммутабельны (immutable).** Это значит, что их значение нельзя изменить после создания. Для `const` это вдвойне верно — нельзя даже переприсвоить саму переменную.

### 4. Выражения (Expressions)

TypeScript достаточно умен, чтобы анализировать не только прямые значения, но и выражения.

**Пример со строкой:**

```typescript
let students: number = 10; // Переменная `students` имеет тип number
let studentAsString: string = students.toString(); // Выражение `students.toString()` возвращает строку.
```

TypeScript понимает, что метод `.toString()` возвращает значение типа `string`, и позволяет присвоить результат переменной с типом `string`.

---

### 5. Тип `number` (Число)

**Что такое число?**
В TypeScript, как и в JavaScript, тип `number` представляет как целые, так и дробные числа. Нет отдельных типов для `int`, `float` и т.д.

**Пример от автора:**

```typescript
let age: number = 30;
let year = 2024; // TypeScript выведет тип `number`.
```

Поведение с `let`/`var` и выводом типов абсолютно идентично строкам.

**Константы с `number`:**

```typescript
const numberOfMembers = 61;
```

TypeScript выведет тип `61` (литеральный числовой тип), а не общий `number`.

**Выражения, возвращающие число:**

```typescript
let stringToNumber: number = parseInt("1985");
```

TypeScript знает, что функция `parseInt` возвращает значение типа `number`, поэтому он позволяет это сделать и корректно выводит тип.

---

### 6. Тип `boolean` (Логический тип)

**Что такое boolean?**
Это тип, который может принимать только два значения: `true` (истина) или `false` (ложь).

**Пример от автора:**

```typescript
let isStudent: boolean = true;
```

Этой переменной можно присвоить только `true` или `false`.

**Важное отличие от других языков:**
В некоторых языках (например, в C) числа `0` и `1` могут использоваться как булевы значения. В TypeScript (и JavaScript) это **не так**.

```typescript
isStudent = 1; // ОШИБКА: Тип 'number' не может быть присвоен типу 'boolean'.
```

**Константы с `boolean`:**

```typescript
const alwaysStudent = true;
```

TypeScript выведет тип `true`, а не общий `boolean`.

**Выражения, возвращающие `boolean`:**
Любое логическое выражение (например, сравнение) будет выведено как `boolean`.

```typescript
let minimumAge: boolean = age >= 6 ? true : false;
// Или даже просто: let minimumAge = age >= 6;
```

TypeScript видит, что выражение `age >= 6` является проверкой, результат которой — `true` или `false`, и выводит тип `boolean` для переменной `minimumAge`.
