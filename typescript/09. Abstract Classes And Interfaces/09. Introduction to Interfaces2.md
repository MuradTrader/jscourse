## Основная идея интерфейсов

### Что такое интерфейс?

**Интерфейс** — это контракт, который заранее определяет структуру объекта или класса. Автор подчеркивает:

> "Interfaces are nothing else but contracts"

**Ключевая метафора:** Интерфейс = Юридический договор

- Он заранее определяет правила и обязательства
- Все стороны должны соблюдать эти правила
- Нарушение контракта приводит к "юридическим последствиям" (ошибкам TypeScript)

## Синтаксис объявления интерфейса

### Базовое объявление

```typescript
interface Person {
  name: string;
  email: string;
  age: number;
}
```

**Что здесь происходит:**

- `interface` — ключевое слово (как `class` или `function`)
- `Person` — имя интерфейса (PascalCase)
- В фигурных скобках — требования контракта

### Применение интерфейса к объекту

```typescript
const person: Person = {
  name: "John",
  email: "john@example.com",
  age: 25,
};
```

**Механика "контракта":**

- `: Person` — указывает, что объект должен соблюдать контракт интерфейса Person
- TypeScript немедленно проверяет соблюдение контракта
- Если свойство отсутствует — ошибка компиляции

## Система контроля TypeScript

### Пример ошибки

```typescript
const person: Person = {}; // ОШИБКА!
```

**Сообщение ошибки:**

> "Type '{}' is missing the following properties from type 'Person': name, email, age"

**Что это значит:**

- TypeScript выступает как "юрист", проверяющий контракт
- Он сравнивает то, что вы написали (объект), с тем, что требуется (интерфейс)
- Находит несоответствия и сообщает о них

## Сравнение с типами (type)

### Два способа достичь одного результата

**Способ 1 — Интерфейс:**

```typescript
interface Person {
  name: string;
  email: string;
  age: number;
}
```

**Способ 2 — Пользовательский тип:**

```typescript
type Person = {
  name: string;
  email: string;
  age: number;
};
```

### Мнение автора о выборе

> "it's your choice and it doesn't matter. What matters is that you do use either of the two"

**Ключевые моменты:**

- Для простых объектов разницы практически нет
- Автор предпочитает интерфейсы для единообразия кода
- Важно использовать КАКОЙ-ТО из способов для типизации

## Расширенные возможности интерфейсов

### Необязательные свойства

```typescript
interface Person {
  name: string;
  email: string;
  age: number;
  phone?: number; // Необязательное свойство
}
```

**Синтаксис:** `?` после имени свойства
**Значение:** Свойство может присутствовать, а может отсутствовать

### Методы в интерфейсах

```typescript
interface Person {
  name: string;
  email: string;
  age: number;
  phone?: number;
  greet?(): void; // Необязательный метод
}
```

**Особенности методов в интерфейсах:**

- Указывается только сигнатура (имя, параметры, возвращаемый тип)
- Реализация пишется в объекте/классе
- `void` — метод ничего не возвращает
- `?` — метод необязательный

## Практическое применение

### Полный пример

```typescript
// Объявление контракта
interface Person {
  name: string;
  email: string;
  age: number;
  phone?: number;
  greet?(): void;
}

// Создание объекта, соблюдающего контракт
const person: Person = {
  name: "Alice",
  email: "alice@example.com",
  age: 30,
  phone: 1234567890,
  greet: function () {
    console.log(`Hello, I'm ${this.name}`);
  },
};
```

## Почему это важно?

### Преимущества использования интерфейсов:

1. **Предсказуемость** — знаете структуру объекта заранее
2. **Безопасность типов** — TypeScript ловит ошибки на этапе разработки
3. **Самодокументирование** — интерфейс показывает, что ожидается
4. **Легкость рефакторинга** — изменения в интерфейсе покажут все места, которые нужно обновить
