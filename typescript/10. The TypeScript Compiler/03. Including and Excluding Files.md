В 95% реальных проектов вы будете использовать вполне определённый набор из 10-15 ключевых опций. Автор обещает сфокусироваться именно на них.

**Текст:**
"Before I jump to the ones that have been already mentioned in the Tsconfig file over here, let me show you two options that exist but have not been mentioned or commented out here. So there is another option available to you that is called exclude."

**Объяснение:**

**Важное замечание:** `tsconfig.json` имеет двухуровневую структуру:

1.  **Опции верхнего уровня** (как `exclude`, `include`, `files`) — управляют тем, _какие файлы_ компилировать.
2.  **Опции компилятора** (внутри блока `"compilerOptions"`) — управляют _как_ компилировать (строгость проверки типов, версия JS и т.д.).

### Детальный разбор опции `exclude`

**Текст:**
"exclude is an array that excludes the files that you don't want your TypeScript compiler to compile within your project root, you can pass the path to the file that you want the TypeScript filer to exclude."

**Объяснение:**
`exclude` — это массив строк. Его задача — создать "чёрный список" для компилятора. Вы перечисляете в нём файлы и папки, которые компилятор должен _игнорировать_, даже если они находятся в корне проекта и имеют расширение `.ts`.

**Текст:**
"So for example, if I want to exclude app2 dot ts file, I can do so like this. Since app2 is in my root directory, I don't need to give the path to the file. It automatically starts from the root directory itself."

**Объяснение:**
Практический пример. Чтобы исключить файл `app2.ts`, который лежит в корне проекта, достаточно указать просто его имя.

```json
{
  "exclude": ["app2.ts"]
}
```

Все пути в `exclude` (и `include`) указываются _относительно корневой директории проекта_, то есть той, где лежит `tsconfig.json`.

**Текст:**
"The setting over here can exclude directories as well and not just files. So for example I can pass my node underscore modules directory over here like so."

**Объяснение:**
`exclude` работает и с целыми директориями. Самый частый пример — исключение папки `node_modules`.

```json
{
  "exclude": ["node_modules"]
}
```

### Критически важный нюанс поведения `exclude`

**Текст:**
"Now, one thing you want to remember about exclude property is that node underscore modules is only excluded if you do not include the exclude property. So if your tsconfig file looks something like this where exclude property is not explicitly defined, then node modules is excluded by default, But the moment you include the exclude property, you have to explicitly exclude the node underscore modules."

**Объяснение:**
Это **самая важная часть видео** о `exclude`. Поведение компилятора кардинально меняется в зависимости от того, объявлена ли у вас опция `exclude`:

1.  **Если `exclude` ОТСУТСТВУЕТ в `tsconfig.json`:**

    - Компилятор работает со списком исключений _по умолчанию_.
    - Этот список по умолчанию включает `node_modules`.
    - Также по умолчанию исключаются файлы из `outDir` (если он указан).

2.  **Если `exclude` ПРИСУТСТВУЕТ в `tsconfig.json`:**
    - Компилятор _полностью игнорирует_ список исключений по умолчанию.
    - Он использует ТОЛЬКО тот массив, который вы указали.
    - **Вывод:** Если вы добавили `exclude` и хотите исключить `node_modules`, вы должны явно прописать его в массиве.

### Использование Wildcards (шаблонов)

**Текст:**
"So we can use a wildcard symbol that is an asterisk followed by any wildcard path that we want to define. So I can say I want to exclude all files with an asterisk followed by exclude TS."

**Объяснение:**
`exclude` поддерживает шаблоны (globs) для более гибкого указания файлов.

- `"*.exclude.ts"` — символ `*` означает "любое имя файла". Этот шаблон исключит все файлы в корне проекта, которые заканчиваются на `.exclude.ts` (например, `test.exclude.ts`, `my-file.exclude.ts`).

**Текст:**
"We can go a step further as well and use double asterisk like so. Now what this tells TypeScript is that inside any directory or sub directory, if you find any file name which has an extension of dot exclude dot ts, do not compile it."

**Объяснение:**
Это более мощный шаблон.

- `"**/*.exclude.ts"` — символ `**/` означает "в этой папке и в любых её подпапках, на любой глубине". Этот шаблон найдет и исключит все файлы с расширением `.exclude.ts` где угодно внутри вашего проекта.

### Детальный разбор опции `include`

**Текст:**
"Just like the exclude property, we also have include property. Include is again an array of values. Now here you can specify if you want to include any specific directories. Once you add this property TypeScript will include only those files and directories that have been mentioned in the include array."

**Объяснение:**
`include` — это противоположность `exclude`. Это "белый список".

- Если опция `include` _не указана_, компилятор ищет все `.ts` файлы в проекте (за вычетом `exclude`).
- Если опция `include` _указана_, компилятор будет обращать внимание **ТОЛЬКО** на те файлы и папки, которые перечислены в этом массиве.

**Пример автора:**

```json
{
  "include": ["src"]
}
```

Эта конфигурация говорит компилятору: "Игнорируй всё, что не лежит в папке `src`. Компилируй только файлы внутри `src` и её подпапок."

---

### Взаимодействие `include` и `exclude`

**Текст:**
"Also always remember how TypeScript treats this. It's always include minus exclude and anything that is not mentioned inside include is not included in the compilation process."

**Объяснение:**
Автор формулирует простое правило для запоминания логики работы компилятора:

**Финальный набор файлов для компиляции = `include` - `exclude`**

1.  Сначала компилятор формирует набор файлов на основе `include` (или всех `.ts` файлов, если `include` не указан).
2.  Затем из этого набора он _вычитает_ всё, что попало под `exclude`.

**Важный вывод:** Если вы используете `include`, всё, что не попало в этот список, автоматически игнорируется.

---

**Текст:**
"Now it turns out that if you want to keep all our files in a single directory like the src direct, then there is a better way to do that which we will look at in the next video."

**Объяснение:**
Автор создаёт интригу. Хотя `include` решает задачу изоляции исходного кода в папке `src`, существует более элегантный и общепринятый способ организации проекта. Скорее всего, в следующем видео пойдёт речь об опции `rootDir` или о настройках для управления выходной директорией (`outDir`).

### Резюме от вашего "профессионального разработчика":

Вы изучили два фундаментальных инструмента контроля файловой структуры проекта:

1.  **`exclude`** — "Чёрный список". Используется для точечного исключения файлов и папок из компиляции. Помните ключевое правило:一旦 вы объявили `exclude`, вы должны явно исключить `node_modules`.

2.  **`include`** — "Белый список". Используется для жёсткого ограничения области компиляции только определёнными папками (например, `src`). Всё, что не в `include`, игнорируется.

**Почему это важно в реальной разработке?**

- **Структура проекта:** Вы организуете код, отделяя исходники (`src`) от скомпилированных файлов (`dist`), тестов (`test`), конфигов и т.д.
- **Производительность:** Компилятор не тратит время на обработку папки `node_modules` или временных файлов.
- **Чистота сборки:** Вы предотвращаете случайную компиляцию служебных или тестовых файлов в продакшн-сборку.

Теперь вы можете осознанно управлять тем, какие части вашего проекта попадают в компилятор. Жду объяснения более эффективного способа организации проекта в следующем видео!
