**Текст:**
"We are back in our Tsconfig file. Let's start by understanding the target property. The target property defines the version of JavaScript to which our TypeScript will be compiled."

**Объяснение:**
**`target`** — это одна из самых важных опций компилятора. Она указывает, в какую версию стандарта ECMAScript (JavaScript) должен компилироваться ваш TypeScript-код.

**Почему это важно?** TypeScript поддерживает современные функции JavaScript (как классы, стрелочные функции, async/await), но не все браузеры и среды выполнения поддерживают эти современные функции. Компилятор TypeScript может преобразовать ваш современный код в более старый синтаксис, который будет работать в старых браузерах или версиях Node.js.

**Текст:**

This means factoring in questions such as Will the TypeScript run in the browser once it is compiled? Are you using React.js with TypeScript or is your project a server side app using node? All these factors impact the target version of JavaScript to which you would want to compile your TypeScript files to."

**Объяснение:**
Автор подчеркивает, что выбор значения `target` — это **стратегическое решение**, которое зависит от контекста вашего проекта:

1.  **Для браузерных приложений:** Нужно учитывать поддержку браузерами. Если вы хотите поддерживать старые браузеры (например, Internet Explorer), вам понадобится более старый `target` (например, `ES5`).

2.  **Для серверных приложений (Node.js):** Вы можете использовать более современный `target` (например, `ES2020`), если знаете, что ваша версия Node.js поддерживает эти функции.

3.  **Для React-приложений:** Create React App по умолчанию использует `ES5` для максимальной совместимости с браузерами.

### Резюме от вашего "профессионального разработчика":

Вы изучили стратегически важную опцию `target`, которая определяет "компиляционную цель" вашего проекта.

**Ключевые выводы:**

1.  **`target` определяет совместимость:** Более старый `target` (как ES5) = лучшая совместимость с браузерами, но более объемный код. Более новый `target` (как ES2022) = более компактный и эффективный код, но требующий современных сред выполнения.

2.  **Используйте проверенные конфигурации:** Не пытайтесь угадать правильные настройки. Берите за основу конфигурации из tsconfig/bases или из популярных boilerplates.

3.  **Контекст решает:** Выбор `target` полностью зависит от того, где будет работать ваш код:

    - **Браузер:** Ориентируйтесь на целевую аудиторию и поддерживаемые браузеры
    - **Node.js:** Можете использовать современные версии, соответствующие вашей версии Node.js

4.  **Практический подход:** Для начала используйте настройки из Create React App или других популярных инструментов. По мере роста опыта вы сможете осознанно менять эти настройки под конкретные нужды.
